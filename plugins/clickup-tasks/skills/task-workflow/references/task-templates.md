# Task Description Templates

Use these templates when creating tasks via `/task:new`.

---

## Feature Task

```markdown
## Summary
[One paragraph describing what this feature does and why it's needed]

## Context
- **Why**: [Motivation - what problem does this solve?]
- **Impact**: [What improves when this is done?]

## Acceptance Criteria
- [ ] [Specific, testable outcome 1]
- [ ] [Specific, testable outcome 2]
- [ ] [Specific, testable outcome 3]
```

---

## Bug Fix

```markdown
## Summary
[Describe the bug - what's happening vs what should happen]

## Steps to Reproduce
1. [Step 1]
2. [Step 2]
3. [Expected result]
4. [Actual result]

## Acceptance Criteria
- [ ] Bug is fixed and no longer reproducible
- [ ] Regression test added
- [ ] No new issues introduced
```

---

## Refactor

```markdown
## Summary
[What is being refactored and why]

## Current State
[Brief description of current implementation]

## Target State
[Description of desired implementation]

## Acceptance Criteria
- [ ] Refactored code meets target state
- [ ] All existing tests pass
- [ ] No regression in functionality
```

---

## Chore

```markdown
## Summary
[What maintenance work needs to be done]

## Tasks
- [ ] [Task 1]
- [ ] [Task 2]

## Acceptance Criteria
- [ ] [Verification that chore is complete]
```

## Minimal (Simple Tasks)

```markdown
## Summary

```
